# Betaflight ADRC Controller (Active Disturbance Rejection Control)

[English](README.md) | **Русский**

Этот репозиторий реализует **Active Disturbance Rejection Control (ADRC)** — активное подавление возмущений — в Betaflight, полностью заменяя классический контур PID. ADRC выступает «убийцей PID»: даёт впечатляющую устойчивость, стойкость к ветру и плавное поведение даже с неоткалиброванными параметрами, при смене размера пропеллеров или с экстремальной несбалансированной динамической нагрузкой.

> ⚠️ **Экспериментальные правки надёжности в этом форке — нужны тестеры-пилоты!**
> Этот форк несёт серию небольших независимых правок надёжности ADRC поверх
> `Boyyt357/ADRC-betaflight` (anti-windup на оценке возмущения, подача в наблюдатель
> зажатой по насыщению команды, пересчитанные под ADRC дефолты, обработка наблюдателя на
> нулевом газе, плюс одна экспериментальная правка). **Они НЕ протестированы на реальном
> железе.** Каждая правка — отдельный коммит, так что можно собрать с любой из них или без
> (`git revert <sha>`). Детали и обоснование — в [`ADRC_FIXES.md`](ADRC_FIXES.md). Если
> полетаешь — пожалуйста, отпишись в
> **[issue #1 — Call for flight testers](https://github.com/danusha2345/ADRC-betaflight/issues/1)**:
> что летал, с какими коммитами и как себя вело. 🙏

## Ключевые особенности
- **Минимум настройки:** отлично летит из коробки даже с грубыми, неоткалиброванными значениями.
- **Несбалансированная нагрузка:** активно оценивает и компенсирует внешние силы на лету, сохраняя стабильный полёт даже с болтающимся грузом на одном луче.
- **Универсальность по пропеллерам:** на лету справляется со сменой размера пропеллеров без изменения параметров.

## Как это работает: переназначение полей PID
Вместо классических коэффициентов Proportional, Integral и Derivative реализация переиспользует поля настройки PID в Betaflight для управления системой ADRC:

| Поле | Параметр ADRC | Описание |
| :---: | :--- | :--- |
| **P** | **Control Bandwidth** (полоса управления) | Задаёт скорость отработки ошибки. Больше — быстрее коррекция; меньше — медленнее. |
| **I** | **Observer Bandwidth** (полоса наблюдателя) | Скорость работы расширенного наблюдателя состояния (ESO): насколько быстро контроллер оценивает и компенсирует внешние силы (ветер, prop wash). *Внимание: слишком высокое значение усиливает шум гироскопа и греет моторы.* |
| **D** | **System Gain** (усиление объекта) | Сообщает контроллеру, насколько мощны моторы (по ускорению и KV). Уменьшение увеличивает общий gain (для быстро разгоняющихся моторов); увеличение снижает gain (для более плавного управления). |

**Настоятельно рекомендуется** отключить PID на минимальном газе на случай, если стартовые параметры ADRC неточны для вашего дрона — иначе при арме он может вести себя непредсказуемо, пока вы подбираете параметры. В командной строке (CLI) Betaflight выполните:
```
set pid_at_min_throttle = off
```

### Примеры параметров
| Тип дрона | Control Bandwidth (P) | Observer Bandwidth (I) | System Gain (D) |
| :--- | :---: | :---: | :---: |
| 10″ дрон (из видео автора) | 10 | 50 | 20 |
| 5″ дрон (jmsweng) | 10 | 110 | 100 |

## Сборка ADRC-Betaflight
Собирается точно так же, как обычный Betaflight (полная документация [здесь](https://betaflight.com/docs/category/building)). На обычном x86_64-хосте (Linux / macOS / WSL):
```
git clone https://github.com/danusha2345/ADRC-betaflight
cd ADRC-betaflight
make arm_sdk_install   # один раз: качает закреплённый arm-none-eabi GCC (13.3.1)
make configs           # один раз: подтягивает сабмодуль с board-конфигами
make DAKEFPVF405       # сборка под вашу плату — замените DAKEFPVF405 на свою
```
Готовый `.hex` появится в `obj/`. (Проверено: чисто собирается для `DAKEFPVF405` / STM32F405 на GCC 13.3.1.)

<details>
<summary>Сборка на ARM-хосте (например, Raspberry Pi)</summary>

Тулчейн, который качает `make arm_sdk_install`, только под x86_64, поэтому на ARM-хосте используйте системный тулчейн. Проверено на Raspberry Pi 3B с Raspbian Trixie 13.5:

1) Установите тулчейн
```
sudo apt update && sudo apt upgrade
sudo apt install gcc-arm-none-eabi libnewlib-arm-none-eabi build-essential
```
2) Склонируйте и войдите в репозиторий
```
git clone https://github.com/danusha2345/ADRC-betaflight
cd ADRC-betaflight
```
3) Закомментируйте строку `$(error No toolchain URL defined ...)` в `mk/tools.mk` (строка 43), чтобы сборка использовала системный тулчейн, а не качала свой.
4) Укажите сборке версию системного компилятора
```
echo "GCC_REQUIRED_VERSION = $(arm-none-eabi-gcc -dumpversion)" >> mk/local.mk
```
5) Подтяните конфиги и соберите
```
make configs
make DAKEFPVF405
```
</details>

## 🧪 Помогите протестировать — нужны тестеры!

Правки надёжности ADRC в этом форке (см. [`ADRC_FIXES.md`](ADRC_FIXES.md)) **НЕ протестированы на реальном железе** — код собирается чисто, но ни разу не летал. Если у вас есть аппарат, на котором можно безопасно тестировать, помогите их проверить.

**Как помочь:**
1. Соберите форк (см. раздел *Сборка* выше). Каждая правка — отдельный коммит, так что через `git revert <sha>` можно собрать с любой из них или без.
2. Тестируйте безопасно — **сначала без пропеллеров**, затем на открытой площадке вдали от людей.
3. Отпишитесь в **[issue #1 — Call for flight testers](https://github.com/danusha2345/ADRC-betaflight/issues/1)**, указав:
   - Аппарат (размер, вес, моторы/пропы, целевая плата) и ваши ADRC P/I/D (wc/wo/b0).
   - С какими коммитами собирали (или «все»).
   - Поведение: арм/раскрутка, висение, резкие манёвры, prop wash, ветер, восстановление после сброса газа, любые осцилляции или нагрев моторов.
   - По возможности — лог blackbox.

Даже короткое «нормально полетел на 5″ со всеми правками» или «поймал осцилляцию по yaw» очень помогает. Спасибо! 🙏

## Проблемы с железом

Betaflight не производит и не распространяет собственное железо. Хотя мы сотрудничаем с рядом производителей и пользуемся их поддержкой, мы не оказываем никакой поддержки по железу.

Если у вас возникли проблемы с полётным контроллером или другим компонентом, обратитесь к производителю или поставщику вашего железа либо загляните в [Discord](https://discord.gg/n4E6ak4u3c) — возможно, кто-то уже решил такую же проблему.
